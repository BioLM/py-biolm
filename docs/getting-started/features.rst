Features
========

Overview of BioLM package features across all interfaces (CLI, SDK, and Protocols).

Core Features
--------------

- **Multiple Interfaces**: Access BioLM via CLI, Python SDK, or Protocol YAML files
- **Authentication**: Flexible authentication options (API tokens, CLI login)
- **Workspaces**: Organize and manage your work
- **Models**: Access to various biological language models (ESM2, ESMFold, ProGen2, etc.)
- **Protocols**: Define and execute workflows using YAML configuration
- **Datasets**: Manage and work with datasets
- **Batch Processing**: Efficient handling of large-scale operations
- **Async Support**: Asynchronous operations for improved performance

Interface-Specific Features
---------------------------

CLI Features
^^^^^^^^^^^^

- Interactive authentication
- Protocol validation
- Configuration management
- Command-line workflows

SDK Features
^^^^^^^^^^^^

- High-level and low-level APIs
- Automatic batching
- Schema-based validation
- Flexible error handling
- Rate limiting
- Disk or memory output
- Async/await support

Protocol Features
^^^^^^^^^^^^^^^^^

- YAML-based configuration
- Schema validation
- Reusable workflow definitions
- Version control friendly
